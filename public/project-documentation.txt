================================================================================
                            1. PROJECT OVERVIEW
================================================================================

This is a comprehensive Global Techno Encyclopedia web application - a curated 
digital archive dedicated to underground techno culture worldwide. The platform 
serves as an authoritative resource covering:

- Artists & DJs from the global techno scene
- Record Labels (iconic and emerging)
- Venues & Clubs (legendary spaces worldwide)
- Festivals (from Berghain to Bassiani territories)
- Gear & Equipment (synthesizers, drum machines, effects)
- Crews & Collectives
- Historical Timeline of techno evolution
- Calendar of events
- News & Editorial content

TARGET AUDIENCE:
- Techno enthusiasts and collectors
- DJs and producers
- Music journalists and researchers
- Club culture historians
- Festival-goers and event organizers

                            5. BACKEND & DATABASE
================================================================================

SUPABASE (via Lovable Cloud)
-----------------------------

DATABASE TABLES:

1. documents
   - id: uuid (PK)
   - title: text
   - content: text
   - source: text (nullable)
   - metadata: jsonb
   - embedding: vector (1536 dimensions)
   - chunk_index: integer
   - created_at: timestamp
   - updated_at: timestamp
   
   Purpose: Stores document chunks for RAG (Retrieval-Augmented Generation)
   RLS: Public read, authenticated write

2. profiles
   - id: uuid (PK)
   - user_id: uuid (FK to auth.users)
   - email: text
   - created_at: timestamp
   - updated_at: timestamp
   
   Purpose: User profile information
   RLS: Users can view/update own profile

3. user_roles
   - id: uuid (PK)
   - user_id: uuid (FK to auth.users)
   - role: app_role enum ('admin' | 'user')
   - created_at: timestamp
   
   Purpose: Role-based access control
   RLS: Users can view own roles

DATABASE FUNCTIONS:

- match_documents(query_embedding, match_threshold, match_count)
  Performs vector similarity search for RAG
  
- has_role(_user_id, _role)
  Checks if user has specific role
  
- handle_new_user()
  Trigger function to create profile on signup

EDGE FUNCTIONS:

1. rag-chat
   - Endpoint for AI-powered chat
   - Retrieves relevant documents via full-text search
   - Generates responses using Lovable AI
   - Supports streaming responses
   
2. generate-embedding
   - Generates vector embeddings for text
   - Uses Lovable AI embeddings API
   
3. ingest-documents
   - Admin-only document ingestion
   - Chunks documents and stores in database
   - Used to populate RAG knowledge base
